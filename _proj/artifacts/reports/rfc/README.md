# RFC Reports

**Category**: `rfc`
**Naming**: `RFC-XXXX-[title].md`

## What Goes Here

- Request for Comments documents
- Design proposals
- Technical proposals
- Feature specifications
- Architecture proposals

## Lifecycle

1. **Draft** — Initial proposal
2. **Review** — Open for feedback
3. **Approved** — Decision made to proceed
4. **Rejected** — Decision made not to proceed
5. **Implemented** — Work completed

## Example Reports

- `RFC-0001-authentication-redesign.md` — Auth system redesign proposal
- `RFC-0002-api-versioning.md` — API versioning strategy
- `RFC-0003-event-sourcing.md` — Event sourcing adoption proposal
