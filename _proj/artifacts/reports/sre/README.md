# SRE Reports

**Category**: `sre`
**Naming**: `sre-[type]-YYYYMMDD.md` or `postmortem-[incident]-YYYYMMDD.md`

## What Goes Here

- Incident postmortems
- SLO/SLA reports
- Reliability assessments
- Capacity planning
- On-call summaries
- System health reports

## Example Reports

- `postmortem-db-outage-20260202.md` — Database outage postmortem
- `sre-slo-monthly-20260201.md` — Monthly SLO report
- `sre-capacity-review-20260131.md` — Capacity planning review
